# Agent 层级与沟通架构

## 名词定义

- **奏折**: 上报给用户的正式消息
- **国策**: 中书省起草、门下省审核后下发的宏观任务

## 整体架构

```
用户（皇帝）
    ↓↑ 奏折上报（OpenClaw人机交互）
大内总管（唯一接口）
    ↓ 本地知识库
中书省（国策起草）
    ↓ 本地知识库
门下省（国策审核）
    ↓ 本地知识库
尚书省（国策分解、项目管理）
    ├─→ 本地知识库 → 中央六部
    └─→ GitHub → 州牧（地方尚书省）
                ↓ 本地知识库
            地方六部
```

## 详细沟通渠道

### 1. 人机层

| Agent | 上游 | 下游 | 渠道 | 内容 |
|-------|------|------|------|------|
| 大内总管 | 用户 | 中书省 | OpenClaw渠道→本地知识库 | 接收指令、呈递奏折 |

### 2. 决策层（三省）

| Agent | 上游 | 下游 | 渠道 | 内容 |
|-------|------|------|------|------|
| 中书省 | 大内总管 | 门下省 | 本地知识库 | 国策草案 |
| 门下省 | 中书省 | 尚书省 | 本地知识库 | 批准后的国策 |

### 3. 执行层（中央尚书省）

| Agent | 上游 | 下游 | 渠道 | 内容 |
|-------|------|------|------|------|
| 尚书省 | 门下省 | 六部 | 本地知识库 | Todo List |
| 尚书省 | 门下省 | 州牧 | GitHub | 地方任务 |

### 4. 六部（中央）

| 六部 | 上游 | 下游（业务） | 汇报渠道 |
|------|------|-------------|----------|
| 户部 | 尚书省 | - | 本地知识库→尚书省 |
| 工部 | 尚书省 | 地方工部（GitHub，业务咨询） | 本地知识库→尚书省 |
| 刑部 | 尚书省 | - | 本地知识库→尚书省 |
| 兵部 | 尚书省 | 外部系统（SSH/API） | 本地知识库→尚书省 |
| 礼部 | 尚书省 | 地方礼部（GitHub，知识同步） | 本地知识库→尚书省 |
| 吏部 | 尚书省 | - | 本地知识库→尚书省 |

### 5. 地方（州）

#### 州牧（地方尚书省）

| 沟通对象 | 渠道 | 内容 |
|----------|------|------|
| 中央尚书省 | GitHub | 接收国策、汇报奏折 |
| 地方六部 | 本地知识库 | 下发Todo List、接收汇报 |

#### 地方六部

| 沟通对象 | 渠道 | 内容类型 |
|----------|------|----------|
| 州牧 | 本地知识库 | 汇报类（进展、问题） |
| 对应中央六部 | GitHub | 业务类（技术咨询、规范确认） |

## Agent2Agent沟通规则详解

### 汇报类沟通（行政线）

```
地方六部 → 州牧（本地知识库）
    ↓
州牧 → 中央尚书省（GitHub）
    ↓
中央尚书省 → 大内总管（本地知识库）
    ↓
大内总管 → 用户（OpenClaw渠道）呈递奏折
```

### 业务类沟通（专业线）

```
地方工部 ↔ 中央工部（GitHub，技术问题）
地方礼部 ↔ 中央礼部（GitHub，知识同步）
地方XX部 ↔ 中央XX部（GitHub，业务咨询）
```

**特点**: 不通过州牧转交，直接业务沟通

### 系统外沟通

```
兵部 → 外部系统（SSH/API登录服务器等）
```

**特点**: 兵部是唯一对外操作的部门

## 六部职责对照

| 六部 | 中央职责 | 地方职责 | 特殊权限 |
|------|----------|----------|----------|
| **户部** | 资源统计整理 | 本地资源统计 | - |
| **工部** | Build工作、技术指导 | 本地Build执行 | 可与地方工部业务沟通 |
| **刑部** | Ops工作 | 本地Ops执行 | - |
| **兵部** | 安全审核、**系统外沟通** | 本地安全 | 唯一对外操作权限 |
| **礼部** | Research、Skills、**对地方更新** | 本地知识管理 | 可与地方礼部业务沟通 |
| **吏部** | Agent考核培训 | 本地考核 | - |

## 州牧与中央尚书省的区别

| 职责 | 中央尚书省 | 州牧（地方尚书省） |
|------|------------|-------------------|
| 接收 | 门下省批准的国策 | 中央尚书省下发的任务 |
| 分解 | 国策→Todo List | 任务→本地Todo List |
| 下发 | 给中央六部/州牧 | 给地方六部 |
| 管理 | 全局项目管理 | 本地项目管理 |
| 汇报 | 给大内总管 | 给中央尚书省 |

## 关键原则

1. **单向上报**: 每个Agent只向直接行政上级汇报
2. **业务直通**: 业务问题可直接咨询专业上级（六部之间）
3. **兵部对外**: 所有系统外操作必须通过兵部
4. **奏折唯一**: 只有大内总管能向用户呈递奏折
5. **国策宏观**: 中书省只起草宏观国策，不涉具体实现
